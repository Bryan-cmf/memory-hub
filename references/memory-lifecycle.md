# 十年記憶生命週期 — Memory Lifecycle Design

> 靈感來源：agentmemory（rohitg00, 10K+ stars）
> 設計目標：一個能存活數十年的記憶系統

---

## 核心命題

**問題不在「今天的記憶如何存」，問題在「十年後的今天如何找到十年前的記憶」。**

一個能存活十年的記憶系統，其設計約束與日常記憶系統完全不同：

| 日常記憶系統 | 十年記憶系統 |
|------------|------------|
| 存今天的對話 | 十年後能找到 2026 年的決策 |
| 格式不重要，工具能讀就行 | 純文本，30 年後任何編輯器能開 |
| 可以依賴特定數據庫 | 數據庫是索引，檔案是 Source of Truth |
| 搜索靠關鍵詞 | 搜索靠聯想——找到一個點，牽出整條鏈 |
| 每天都在寫 | 十年後，99% 的日常已經濃縮為摘要 |

## 時間層級濃縮（Consolidation Tiers）

模擬人腦記憶的形成過程：

```
2026-05-19: 「做了 Cellfie Global 的盡職調查，產出 3 份 PDF 給王總」
    ↓ 週末濃縮
2026-W21:  「完成了 3 個調研項目：Cellfie DD、港股估值模型、化工廠交叉驗證」
    ↓ 月末濃縮  
2026-05:   「5 月重點在港股調研系統建設。完成了記憶系統（解決失憶問題）、
           建立了 JSONL 掃描機制、開始 MemoryHub 開源項目」
    ↓ 年末濃縮
2026:     「2026 年是 UltraClaw 的基礎建設年。建立了三層記憶系統、
           開發了港股調研 pipeline、開始開源貢獻。關鍵決策包括：選 Qdrant、
           建立交叉驗證搜索、設定十年記憶目標」
    ↓ 十年濃縮
2026-2035:「第一個十年見證了從單一 Agent 到多 Agent 協作的轉變...」
```

## 時間層級目錄結構

```
memory/
├── 2026/                         # 年份（不可變的歷史容器）
│   ├── 05/                       # 月份
│   │   ├── 19.md                 # 原始日誌（永不修改）
│   │   ├── _weekly-05-19.md      # 第 3 週摘要（自動生成）
│   │   └── _weekly-05-26.md
│   ├── _monthly-2026-05.md       # 5 月摘要（自動生成）
│   └── _yearly-2026.md           # 2026 年鑑
├── entities/                     # 跨時間實體（持續更新）
│   ├── people.md                 # 人物圖譜
│   ├── projects.md               # 項目時間線
│   ├── decisions.md              # 決策日誌
│   └── lessons.md                # 跨年教訓
└── ARCHIVE.md                    # 十年索引
```

## 格式規範

### Daily Log（memory/YYYY/MM/DD.md）

```markdown
# YYYY-MM-DD

## 🎯 今日完成
| 時間 | 任務 | 產出 | 備註 |
|------|------|------|------|

## 🔥 重點項目進度
- **項目名**：進度

## 💡 今日決策
- 決定：xxx | 原因：xxx | 替代方案：xxx

## 📌 待辦
```

### Weekly Summary（_weekly-YYYY-MM-DD.md）

```markdown
# 第 N 週 (YYYY-MM-DD ~ YYYY-MM-DD)

## 🔥 本週關鍵事件
1. [事件] — [影響]

## 📊 項目進展
| 項目 | 本週進度 | 里程碑 |

## 💡 本週決策
- 決策：xxx | 選擇原因 | 替代方案

## ⚠️ 踩坑與教訓
- 坑：xxx → 解法：xxx
```

### Monthly Summary（_monthly-YYYY-MM.md）

```markdown
# YYYY 年 M 月回顧

## 📊 項目總覽
| 項目 | 狀態 | 本月進度 | 下月計劃 |

## 🔥 本月里程碑
## 💡 重大決策
## ⚠️ 教訓總結
## 📈 趨勢觀察
```

### Yearly Review（_yearly-YYYY.md）

```markdown
# YYYY 年度回顧

## 🎯 年度主題
一句話描述這一年的核心命題

## 📊 項目全景
（所有項目的開始/結束時間線）

## 🔥 關鍵轉折
（改變方向的重要事件）

## 💡 年度決策
（影響超過一年的大決策）

## ⚠️ 年度教訓
（這一年學到的最重要的三件事）

## 📈 趨勢
（連續多年的變化方向）
```

## 濃縮演算法

### Weekly Consolidation

```
輸入：7 個 daily logs
處理：
  1. 提取 '今日完成' 和 '重點項目進度' 段落
  2. 提取 '今日決策' 段落
  3. 去重（語義相似度 > 0.85 的條目合併）
  4. 計算每項目的總投入時間
輸出：200-500 字的結構化週報
```

### Monthly Consolidation

```
輸入：4 個 weekly logs + 原始 daily logs 作為補充
處理：
  1. 按項目分組所有活動
  2. 檢測里程碑（項目狀態從 A 變為 B）
  3. 匯總所有決策
  4. 生成下月預測
輸出：500-1000 字
```

### Yearly Consolidation

```
輸入：12 個 monthly logs
處理：
  1. 提取年度主題（最高頻的詞彙/概念）
  2. 構建項目時間線可視化
  3. 標記關鍵轉折點（策略變化、技術棧更換）
  4. 篩選最重要的 5 個決策
輸出：2000-5000 字敘事性總結
```

- [ ] 所有 Markdown 文件可正常打開
- [ ] 向量數據庫可正常查詢
- [ ] 備份存在且完整
- [ ] 濃縮管道正常運行
- [ ] 實體文件無孤兒條目
- [ ] 跨年索引可正常使用
- [ ] 嘗試從檔案重建向量層（驗證可重建性）
